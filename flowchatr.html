Flowchart between Player, KZ and 3rd Payment Provider

Flow 1: Create order and send from KZ

%%{init: { "themeVariables": { "actorBkg": "#d0e9ff", "actorBorder": "#7fbfff" }}}%% sequenceDiagram participant Player as Player participant KZ as KZ participant Provider as 3rd Payment Provider Note over Player,KZ: Player A open KZ, click create order Player->>KZ: Send request create order KZ->>KZ: Create order, construct payload KZ->>Provider: Send request Provider->>KZ: Response data (with account number: CLABE) KZ->>Player: Return CLABE Player->>Player: Copy CLABE, open app bank Player->>Provider: Transfer fund to CLABE Provider->>Provider: Verify fund Provider-->>KZ: Callback to notify fund received Note right of KZ: Callback/validation step
Sometimes later, player want to send more fund but do not need to open KZ again

Flow 2: Player transfers fund DIRECTLY through app bank

sequenceDiagram participant Player as Player participant KZ as KZ participant Provider as 3rd Payment Provider Player->>Provider: Transfer directly to CLABE (not open KZ) Provider->>Provider: Accept, verify fund, create order Provider-->>KZ: Callback notify fund received Note right of KZ: Callback/validation step KZ->>KZ: Verify order callback alt True KZ->>KZ: Create order KZ, credit fund to player's wallet else False KZ-->>KZ: NO create order end
